## Design Principles

### 1. Excel Terminology, Not SQL

| Technical Term | User-Friendly Label | Tooltip |
|----------------|---------------------|---------|
| WHERE clause | **Filter** | "Keep only certain rows" |
| SELECT columns | **Select Columns** | "Choose which columns to keep" |
| JOIN | **Combine Tables** | "Bring in data from another table (like VLOOKUP)" |
| GROUP BY | **Summarize** | "Create a summary (like a Pivot Table)" |
| ORDER BY | **Sort** | "Arrange rows in order" |
| DISTINCT | **Remove Duplicates** | "Keep only unique rows" |
| CASE WHEN | **Categorize** | "Create categories from values" |
| String functions | **Clean Data** | "Fix messy text" |
| UNION | **Stack Tables** | "Put tables on top of each other" |

### 2. No Code Visible by Default

- SQL is hidden completely from the main interface
- Available only via "Advanced > View SQL" menu
- Users never need to see or understand SQL

### 3. Instant Feedback

- Row counts update as steps are added
- Preview shows first 100 rows immediately
- Progress bars show percentage of data affected

### 4. Guided Workflows

- Each action opens a modal with clear instructions
- Tooltips explain each option
- Preview shows result before applying

### 5. Familiar Layout

- Spreadsheet grid (like Excel)
- Sidebar with steps (like Power Query's Applied Steps)
- Action buttons (like Excel's Data tab)

## Component Specifications

### Data Grid (AG Grid)

**Configuration:**
```tsx
<AgGridReact
  rowModelType="infinite"          // Server-side pagination
  datasource={serverSideDatasource}
  cacheBlockSize={100}             // Fetch 100 rows at a time
  maxBlocksInCache={10}            // Keep 1000 rows in memory
  columnDefs={columnDefs}
  defaultColDef={{
    sortable: true,
    resizable: true,
    filter: false,                 // Filtering done via our modal
  }}
/>
```

**Features used:**
- Infinite scroll (server-side pagination)
- Column sorting
- Column resizing
- Row selection (for debugging)

**Features NOT used (to keep simple):**
- Client-side filtering (use our Filter modal)
- Cell editing (read-only preview)
- Row grouping (use our Summarize modal)

### Steps Panel

```typescript
interface Step {
  id: string;
  type: 'source' | 'filter' | 'join' | 'group' | 'sort' | 'clean' | 'select' | 'categorize' | 'distinct';

interface TransformationState {
  steps: Step[];

  totalRows: number;

```
